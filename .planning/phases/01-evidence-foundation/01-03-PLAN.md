<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Committed seed validation to Vite startup | The browser must not show unvalidated civic material when the seed is invalid. |
| Static evidence module to React detail panel | Malformed or incomplete projection data must not become a claim. |
| React citation control to browser navigation/clipboard | Only stored public links and readable citations may be exposed. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-03-01 | Tampering | demo startup | high | mitigate | Make npm run demo execute the read-only DB validator before starting Vite and fail non-zero on diagnostics. |
| T-03-02 | Information disclosure | client bundle | high | mitigate | Import only src/demo-seed.ts and types; never bundle SQLite, raw evidence, provider configuration, or secrets. |
| T-03-03 | Spoofing | public-record action | medium | mitigate | Render the stored validated canonical URL only and use readable stored source labels. |
| T-03-04 | Denial of service | malformed read model | low | mitigate | Render explicit error/empty branches and avoid implicit selection or automatic rebuild/fetch behavior. |
</threat_model>
