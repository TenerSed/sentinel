<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Hand-verified fixture to SQLite seed | Human-entered civic facts can be incomplete, wrong, or lack provenance. |
| SQLite seed to static browser projection | Only eligible citation-bearing rows may cross into the Vite bundle. |
| Vite browser to public source link | Stored canonical URLs must not become an arbitrary navigation vector. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-01-01 | Tampering | fixture-to-seed provenance | high | mitigate | Reject invalid quotes, URLs, locators, and foreign keys in the shared validator before projection. |
| T-01-02 | Information disclosure | Vite import graph | high | mitigate | Keep SQLite in .mjs Node scripts and export a narrow static read model only. |
| T-01-03 | Spoofing | public-record link | medium | mitigate | Require normalized HTTPS canonical URLs from configured sources before an evidence record is accepted. |
| T-01-SC | Tampering | npm dependency install | high | mitigate | Blocking human package-legitimacy verification before installing the native SQLite package. |
</threat_model>
