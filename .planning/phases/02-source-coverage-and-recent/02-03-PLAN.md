## Trust Boundaries

| Boundary | Description |
|---|---|
| Static seed to React | The UI must not display a record outside validated coverage or introduce a new factual claim. |
| Feed metadata to user interpretation | Dates, update types, and reporting labels must not imply government/legal status. |
| Citation action to browser | The source link must be the stored canonical public URL, unchanged. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-02-08 | Tampering | client filtering | high | mitigate | Use generated coverage data, retain seed validation, and test with the scope check before build. |
| T-02-09 | Spoofing | update/reporting presentation | medium | mitigate | Textual update and Reporting/primary-record labels plus no status language. |
| T-02-10 | Information disclosure | evidence panel | medium | mitigate | Render only narrow record fields, never raw assets/import DB/secrets. |
| T-02-11 | Accessibility denial | fixed navigation | low | mitigate | Semantic controls, 44px targets, focus visibility, and content bottom padding. |
</threat_model>
