# Fishers raw extraction

`npm run ingest:cc -- --backfill --limit=5`, `npm run ingest:parcels -- --limit=2000`, `npm run ingest:yt -- --limit=3`, `npm run ingest:yt -- --video=<YouTube-id>` (a one-video check), `npm run ingest:zoning`, and `npm run ingest:all -- --backfill` write only to `data/fishers.db`. Omit `--live` for backfill; `--live` uses the CivicClerk maximum stored start time and otherwise re-upserts the current source snapshot.

CivicClerk events use `/Events`; files use `/Events/GetEventFileStream(fileId=...,fileType=...,plainText=true)`, falling back to `/Meetings/GetMeetingFileStream(fileId=...,plainText=true)` and then the PDF stream. Structured agenda-items and votes are not exposed by the public CivicClerk API: all `/Meetings` endpoints return 404, so those details remain available only in extracted Minutes and Packet text.

Required system binaries: Poppler's `pdftotext` (`brew install poppler`) extracts PDF text on this Node 18 runtime; `yt-dlp` (installed in `.venv/bin/yt-dlp`) fetches YouTube auto-captions. One minutes file is a scanned image PDF and needs OCR, which this pipeline intentionally does not perform. YouTube auto-captions can still be disabled, throttled, or unavailable, and unavailable videos do not abort a run. Zoning comes from Fishers' `maps.fishers.in.us` ArcGIS service: its district/overlay polygons are dissolved by class, appropriate for a district or corridor view. For per-parcel zoning in the future, use the [Fishers open-data hub](https://data-tofgis.opendata.arcgis.com). Parcel assessor pages are not scraped; `PROPERTYREPORT` is stored as supplied.
